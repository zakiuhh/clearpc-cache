Before You Run

Read this first.

The script is safe by design — but here's what to know going in.

Windows.old deletion is permanent

If C:\Windows.old exists, it will be permanently removed using takeown + icacls. This eliminates the ability to roll back to a previous Windows version. If the folder doesn't exist on your PC, it's automatically skipped.

DISM phase takes 5–15 minutes

Phase 7 runs DISM against the WinSxS component store. This is real processing time that varies by system age and update history. Do not close the window mid-run or the component store may be left in an inconsistent state.

Close browsers for a complete clean

Browser cache files are locked by running processes. If Chrome or Firefox is open, those cache files are silently skipped. Close all browser windows before running the script for a 100% browser cache wipe.

Windows Update keeps working normally

Clearing SoftwareDistribution does not break Windows Update. It simply re-downloads required files on its next check. All 8 services are fully restored by the script before it exits.

FAQ

Common questions.

Everything you need to know before running the script on your machine.

Will this break my Windows installation?
No. The script only deletes cache, log, and temporary files — not system binaries, registry entries, or user data. All services paused during cleaning are fully restored. Windows automatically recreates all these files on demand.
How much disk space will I recover?
It depends on the system. On a regularly maintained PC: 500 MB – 2 GB. On a neglected system: 2 GB – 10 GB. If Windows.old or $Windows.~BT are present: an extra 10 GB – 30 GB+. The WinSxS DISM cleanup typically frees another 1–6 GB.
Will my passwords, bookmarks, and history be deleted?
No. The script only targets cache subfolders (Cache, GPUCache, Code Cache). It does not touch browser profile data — your passwords, history, bookmarks, extensions, cookies, and saved form data are completely untouched.
Why did my taskbar and desktop icons disappear for a second?
That's Phase 8 in action. The script kills and restarts explorer.exe to flush the icon and thumbnail cache. The taskbar and desktop icons disappear for 1–3 seconds while Explorer relaunches — this is completely expected behavior, not an error.
Why does Phase 7 (DISM) take so long?
DISM scans the entire WinSxS component store, which can hold hundreds of old update packages depending on how long your system has been running. Older systems with more accumulated updates take longer. It's doing real work — just let it finish without interruption.
Is it safe to run on a work computer?
For personal PCs with full admin access — yes, completely safe. For managed enterprise machines — check with your IT department first. The script clears Windows Event Logs and Defender scan history, which IT teams may monitor. On a home PC it is safe to run freely.
How often should I run this?
Once a month is sufficient for most users. Developers or power users who install and uninstall software frequently may benefit from running it every 2–3 weeks. There is no harm in running it more often — it simply finds less to delete each time.
Does it work on Windows 7 or Windows 8?
No — the script is designed and tested for Windows 10 and Windows 11 only. Commands like wsreset.exe, certain DISM flags, and several cache paths do not exist on older Windows versions.
